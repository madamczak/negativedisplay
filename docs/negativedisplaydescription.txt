==============================================
NEGATIVE PHOTO DISPLAY APP - AGENTIC WORKFLOW
==============================================

PROJECT OVERVIEW:
Android application for displaying negative photographs on paper for photographic printing purposes.
The app converts photos to negatives and displays them with precise timing controls for paper exposure.

==============================================
PHASE 1: PROJECT SETUP & ARCHITECTURE
==============================================

AGENT TASK 1.1: Initialize Android Project
- Create new Android project with Kotlin
- Minimum SDK: API 24 (Android 7.0)
- Target SDK: Latest stable
- Use ViewBinding for UI interactions
- Dependencies needed:
  * androidx.appcompat
  * androidx.constraintlayout
  * Material Design Components
  * Kotlin Coroutines for async operations
  * AndroidX Activity Result API for photo selection

AGENT TASK 1.2: Define App Architecture
- Use MVVM architecture pattern
- Create following packages:
  * ui (Activities, Fragments, ViewModels)
  * data (Models, Repositories)
  * utils (Image processing, Sound management)
  * domain (Use cases/business logic)

==============================================
PHASE 2: DATA MODEL & BUSINESS LOGIC
==============================================

AGENT TASK 2.1: Create Data Models
- PhotoModel class:
  * uri: Uri (original photo location)
  * bitmap: Bitmap? (loaded image)
  * negativeBitmap: Bitmap? (converted negative)
  * isNegativeConverted: Boolean

- AppSettings class:
  * preDisplayBlackSeconds: Int (X value)
  * displayDurationSeconds: Int (Y value)
  * postDisplayBlackSeconds: Int (Z value)
  * testIrradiationParts: Int (A value)
  * isInterfaceRed: Boolean

AGENT TASK 2.2: Create Repository Classes
- PhotoRepository:
  * Manage list of loaded photos
  * Track current photo index
  * Provide methods: addPhotos(), getCurrentPhoto(), nextPhoto(), previousPhoto()

- SettingsRepository:
  * Store and retrieve X, Y, Z, A values
  * Store interface color state

AGENT TASK 2.3: Image Processing Utility
- ImageProcessor class with methods:
  * convertToNegative(bitmap: Bitmap): Bitmap
    - Iterate through pixels
    - Invert RGB values: newR = 255 - oldR, newG = 255 - oldG, newB = 255 - oldB
  * createTestIrradiationImage(bitmap: Bitmap, parts: Int): List<Bitmap>
    - Divide image width into A parts
    - Create sequence of bitmaps showing progressive reveals
    - Part 1 shows leftmost section, Part 2 shows 2 leftmost sections, etc.

PHASE 8: DISPLAY NEGATIVE FUNCTIONALITY
==============================================

AGENT TASK 8.1: Display Negative Workflow
- When "Display Negative" button clicked:

  2. Get X, Y, Z values from settings
  3. Pass negative bitmap and timing values to FullscreenDisplayActivity
  4. Start activity with Intent extras

AGENT TASK 8.2: Fullscreen Display Sequence Implementation
- FullscreenDisplayActivity lifecycle:
  1. onCreate():
     - Hide system UI completely
     - Set screen to maximum brightness
     - Display pure black screen

  2. Timed sequence using Kotlin Coroutines:
     ```
     launch {
         // Phase 1: Pre-display black (X seconds)
         displayBlack()
         delay(X * 1000L)

         // Phase 2: Display negative (Y seconds)
         playStartSound()
         displayNegativeImage()
         delay(Y * 1000L)
         playEndSound()

         // Phase 3: Post-display black (Z seconds)
         displayBlack()
         delay(Z * 1000L)

         // Finish and return to main screen
         finish()
     }
     ```

  3. Prevent user interruption during sequence (disable back button)
  4. Handle screen rotation lock (portrait or landscape based on config)

PHASE 9: TEST IRRADIATION FUNCTIONALITY
==============================================

AGENT TASK 9.1: Test Irradiation Logic
- When "Test Irradiation" button clicked:
  1. Validate: negative exists AND interface is red
  2. Get Y value (display duration) and A value (number of parts)
  3. Generate progressive reveal bitmaps:
     - Divide image width into A parts
     - Create A bitmaps where bitmap[i] shows leftmost (i+1) sections
  4. Calculate timing: Each part displays for 1 second

AGENT TASK 9.2: Progressive Reveal Algorithm
- ImageProcessor.createTestIrradiationBitmaps(negativeBitmap, parts):
  * Calculate partWidth = bitmap.width / parts
  * For i in 1 to parts:
    - Create new bitmap
    - Copy leftmost (i * partWidth) pixels from negative
    - Fill remaining pixels with black
    - Add to list
  * Return List<Bitmap>

AGENT TASK 9.3: Test Irradiation Display Sequence
- FullscreenDisplayActivity with test mode:
  ```
  launch {
      // Phase 1: Pre-display black (X seconds)
      displayBlack()
      delay(X * 1000L)
      playStartSound()

      // Phase 2: Progressive display (A seconds total)
      for (i in 0 until testBitmaps.size) {
          displayImage(testBitmaps[i])
          delay(1000L) // 1 second per part
      }
      playEndSound()

      // Phase 3: Post-display black (Z seconds)
      displayBlack()
      delay(Z * 1000L)

      finish()
  }
  ```

AGENT TASK 9.4: Test Results Interpretation

  * Part 1 (leftmost): Y seconds exposure
  * Part 2: (Y-1) seconds exposure
  * ...
  * Part A (rightmost): (Y-A+1) seconds exposure
- Add informational dialog explaining the test strip results

CORE CLASSES TO IMPLEMENT:
1. MainActivity - Main UI and photo management
2. MainViewModel - Business logic and state management
3. FullscreenDisplayActivity - Display controller
4. PhotoRepository - Photo data management
5. ImageProcessor - Image conversion utilities
6. ScreenController - Screen state management
7. SoundManager - Audio feedback
8. SettingsRepository - App settings persistence

KEY ALGORITHMS:
1. Negative Conversion:
   - For each pixel: RGB_new = (255 - R, 255 - G, 255 - B)

2. Progressive Reveal:
   - partWidth = imageWidth / numParts
   - bitmap[i] shows leftmost (i+1) * partWidth pixels
   - Exposure time for part i = (totalTime - i) seconds

3. Timing Sequence:
   - Black screen (X sec) → Sound → Image display (Y sec) → Sound → Black screen (Z sec)

CRITICAL REQUIREMENTS:
- Completely hide system UI during display (no navigation buttons)
- Maximize screen brightness during exposure
- Accurate timing (use Coroutines with delay for precision)
- Prevent accidental activation (require negative AND red interface)
- Audio feedback for start/end of exposure
